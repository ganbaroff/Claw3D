### 3. Общий мозг для агентов
- Каждый агент при каждом запросе видит: `session-context.md` + `cto-kanban.md` + `AGILE-RULES.md`
- После каждого audit-цикла `swarm-synthesizer` обновляет `session-context.md`
- Агенты не стартуют с нуля — знают состояние проекта

### 4. Автономный режим → Event-driven
**Был:** cron каждые 2 часа (тратил токены впустую)
**Стал:** агенты idle, просыпаются только на реальные события

```
Railway/GitHub/Sentry → POST /webhook → classifyEvent() → wakeAgent() → finding → kanban
```

- `POST /webhook` — принимает события, верифицирует HMAC подпись
- `POST /event` — внутренний триггер (office UI / тесты)
- `classifyEvent()` — автоматически определяет домен по содержанию события
- `wakeAgent()` — агент разбирается, пишет finding, добавляет задачу в канбан
- Ежедневный digest в полночь — обновляет общий мозг

**Проверено live:** `sentry:error.unhandled → security-agent проснулся за 2с → нашёл дыру JWT → написал готовый код фикса → Z-EV-MNMVBDDE`

## Life Simulator — что это

*Из ROADMAP.md и agent-state-model-spec.md + слова команды:*

**claw3d** — 3D офис (Next.js + Three.js). Агенты визуализированы как аватары (Ready Player Me).

**Life Simulator** — следующий слой поверх офиса:
- Агенты имеют **видимые состояния**: `idle` / `focused` / `working` / `blocked` / `recovering`
- Состояния отражают реальную активность: получил событие → `working`, долго молчит → `idle`/sleeping
- Аватар подходит к доске когда активен, "засыпает" когда idle
- Это не игра — это **визуализация координации**. Пользователь видит кто занят, кто свободен, куда роутить задачу
- Ready Player Me аватары — для персонализации каждого агента

**Что готово в claw3d сейчас:**
- 3D офис задеплоен на Railway
- Агенты отображаются в офисе
- WS соединение с gateway (`NEXT_PUBLIC_GATEWAY_URL`)

**Что не готово (Z-02, Z-03):**
- Z-02: ответы агентов не отражаются в облаке (RemoteAgentChatPanel)
- Z-03: Life Sim / Ready Player Me аватары не реализованы
- Z-05: per-session memory для агентов (теперь есть через `memory/users/`)
- Z-06: autonomous coordinator mode (теперь есть через event-driven system)
