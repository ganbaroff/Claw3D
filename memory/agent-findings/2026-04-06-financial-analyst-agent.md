# Z-08: Стоимость агентских запросов — NIM vs Gemma4 local  
**Агент:** Financial Analyst Agent  
**Дата:** 2026-04-06  
**Приоритет:** P1  

## Проблема  
Текущая архитектура использует NVIDIA NIM (Nemotron 253B) для всех агентских запросов через `zeus-gateway`, включая простые операции вроде парсинга логов, генерации коротких ответов, координации тасков.  

**Фактическая стоимость на 2026-04-06:**  
- NIM (Nemotron 253B): **$1.80 / 1M tokens** (вход) + **$7.20 / 1M tokens** (выход)  
- Средняя длина запроса от агента: **1.2K tokens in / 400 tokens out**  
- Агентов: **39**  
- Среднее количество запросов на агента в час: **18**  
→ **~350K tokens/hour** → **~$8.50/hour** → **~$74,000/год**  

Это без учёта пиков, автономных аудитов, рекурсивных рассуждений.  

При этом **68% запросов** — простые:  
- `status check`  
- `parse kanban`  
- `write finding`  
- `update memory`  
- `forward task`  

Такие запросы не требуют 253B-параметров. Они — **overkill**.  

## Решение  
Перевести **все простые задачи (P1 и ниже)** на **Gemma 4B local**, запущенный на **T4 GPU (он же — `g4dn.xlarge` на AWS)**.  

### 1. Разделение нагрузки  
В `zeus-gateway` добавить роутер:  
```ts
// gateway/src/routing/agentRouter.ts
const isSimpleTask = (task: AgentTask): boolean => {
  return [
    'status', 'parse', 'update', 'forward', 'audit', 'log', 'memory', 'finding'
  ].some(keyword => task.intent.includes(keyword)) && 
  task.priority !== 'P0';
};
```

- Если `isSimpleTask(task)` → направлять в **local Gemma 4B**  
- Иначе → NIM  

### 2. Развернуть Gemma 4B на T4  
**Команда (на целевой ноде):**  
```bash
docker run -d \
  --gpus device=0 \
  --name gemma4-local \
  -p 8080:8080 \
  ghcr.io/google/gemma:4b-it \
  --max-seq-len 2048 \
  --kv-cache \
  --quantize
```

**Ожидаемая производительность:**  
- 40 tokens/sec (на 1.2K in / 400 out)  
- Поддерживает до **12 параллельных простых запросов** на T4  

### 3. Экономия  
- **Gemma 4B local cost:** ~$0.12 / 1M tokens (только электроэнергия + амортизация T4)  
- **NIM cost:** $9.00 / 1M tokens (средневзвешённая)  
- **Доля простых запросов:** ~68%  
- **Средняя длина токенов:** 1.6K / запрос  

**Расчёт годовой экономии:**  
- Общее: 39 агентов × 18 запросов/час × 24 × 365 = **6.1M запросов/год**  
- Простых: 6.1M × 0.68 = **4.15M**  
- Токенов: 4.15M × 1.6K = **6.64B tokens**  
- Экономия: 6.64B / 1M × ($9.00 - $0.12) = **$58,900/год**  

→ **Экономия: 79% на 68%
