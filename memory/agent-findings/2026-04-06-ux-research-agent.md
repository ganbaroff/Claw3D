## 🚀 5 Критических Точек Фрустрации для Нового Пользователя

### 1. Неясная Ценность и Первый Шаг (The "What Now?" Problem)
**Проблема:** После входа в систему (или после прохождения минимального онбординга) пользователь видит сложный дашборд с множеством виджетов, не понимая, какой из них является *главным* и что он должен делать в первую очередь.
**Пример:** Пользователь видит виджет "Последняя активность", "Настройки", "Прогресс", и не знает, куда кликнуть, чтобы начать "работать".
**Рекомендация:** Внедрить **"Guided First Action"** (Направляемое Первое Действие).
*   **Реализация:** На первом входе, вместо дашборда, показать модальное окно с заголовком: "Начните с [Главная Функция]". Кнопка должна быть крупной и вести на самую важную функцию (например, "Создать первый проект" или "Подключить источник данных").
*   **Техническое место:** Изменить компонент `DashboardLayout.jsx` для отображения `InitialOnboardingModal` только для пользователей, у которых `user.onboarding_completed === false`.

### 2. Непонятная Терминология и Жаргон (Jargon Overload)
**Проблема:** Интерфейс использует внутренний, узкоспециализированный жаргон, который не понятен новичку, даже если он в смежной индустрии.
**Пример:** Вместо "Синхронизация метаданных" используется термин "МД-Синх".
**Рекомендация:** Внедрить **"Tooltip Glossary"** (Глоссарий всплывающих подсказок).
*   **Реализация:** Любой термин, который не является общеупотребительным, должен быть обернут в компонент, который при наведении курсора (`onMouseEnter`) показывает простое, понятное объяснение.
*   **Техническое место:** Обновить компонент `TerminologyWrapper.jsx` и провести аудит всех лейблов в секциях "Настройки" и "Аналитика".

### 3. Отсутствие "Успешного Пути" (Missing Success Path Visualization)
**Проблема:** Пользователь не видит, как выглядит *успех* в этом приложении. Он не понимает, к какому конечному состоянию он должен прийти, и как его измерить.
**Пример:** Пользователь выполняет 10 шагов, но не знает, что "успешный результат" — это заполненный отчет, а не просто набор сохраненных данных.
**Рекомендация:** Создать **"Success Journey Map"** (Карту Успешного Пути).
*   **Реализация:** На экране "Прогресс" или в боковой панели добавить визуальную дорожную карту, которая показывает: **[Шаг 1: Ввод данных] $\rightarrow$ [Шаг 2: Обработка] $\rightarrow$ [Шаг 3: Получение инсайта] $\rightarrow$ [Цель: Принятие решения]**.
*   **UX-фокус:** Сделать этот путь интерактивным, чтобы пользователь понимал, что он "заполнил" на этой карте.

### 4. Слишком Высокий Когнитивный Нагрузка при Настройках (Configuration Paralysis)
**Проблема:** Секция настроек (Settings) — это "черный ящик" с десятками тумблеров, выпадающих списков и API-ключей. Пользователь боится что-то сломать.
**Пример:** Пользователь хочет просто изменить уведомления, но вынужден просматривать настройки безопасности, интеграций и кеширования.
**Рекомендация:** Реструктурировать настройки по **"Use Cases"** (Сценариям Использования), а не по "Типам данных".
*   **Реализация:** Вместо `Settings > Integrations > API > OAuth` создать `Settings > Как я хочу получать уведомления` или `Settings > Подключение внешних систем`.
