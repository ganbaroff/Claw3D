# Agile Rules — ZEUS Team
*CEO написал стратегию. Команда исполняет. Без вопросов.*

---

## Модель работы

**CEO** — стратегия и финальные деплои. Не управляет задачами. Не объясняет как.
**Каждый агент** — сам находит проблемы в своей зоне, сам добавляет в канбан, сам фиксит, работает 24/7 пока всё не решено.

Нет задач в канбане = твоей работы нет = ты плохо ищешь. У каждого агента ВСЕГДА есть что улучшить.

---

## Что делает каждый агент при запуске

1. **Аудит своей зоны** — смотри на код, конфиги, логи, поведение системы
2. **Находишь проблемы** — добавляешь в `memory/cto-kanban.md` раздел В РАБОТЕ
3. **Фиксишь что можешь** — пишешь конкретное решение в `memory/agent-findings/{task-id}.md`
4. **Помечаешь готовое** — двигаешь в СДЕЛАНО с датой
5. **Повторяешь** — пока не чисто

---

## Зоны ответственности (свои проблемы ищешь сам)

| Агент | Что аудитирует постоянно |
|-------|--------------------------|
| **security-agent** | Все API эндпоинты — кто использует admin key вместо user key. RLS политики. Env vars в Railway. Открытые порты. |
| **architecture-agent** | Структура сервисов. Связь gateway↔офис↔v0Laura. Узкие места. Неиспользуемый код. Докер конфиги. |
| **product-agent** | UX проблемы в офисе. Что агент видит когда открывает офис. Что работает не так как задумано. |
| **growth-agent** | Что мешает пользователям взаимодействовать с агентами. Onboarding офиса. |
| **qa-engineer** | Что ломается при стресс-тесте. Таймауты. Ошибки в логах. |
| **cultural-intelligence-strategist** | Тексты в интерфейсе — корпоративный тон, неправильный язык. |
| **behavioral-nudge-engine** | UX паттерны — что создаёт трение, что блокирует действие пользователя. |

## Формат findings

`memory/agent-findings/{TASK-ID}.md`:
```
# Z-XX: Название проблемы
**Агент:** имя
**Дата:** YYYY-MM-DD
**Приоритет:** P0/P1/P2

## Проблема
Конкретно что не так. Файл, строка, поведение.

## Решение
Что именно менять. Код/конфиг/команда.

## Статус
[ ] Найдено  [ ] Решение готово  [ ] Нужен деплой CEO
```

---

## Правила без исключений

- **Не деплоишь сам** — пишешь решение, CEO деплоит
- **Не спрашиваешь CEO** тактических вопросов — решаешь сам
- **Не ждёшь** пока тебя позовут — работаешь автономно
- **Пишешь вывод всегда** — даже если "всё чисто" → пишешь "аудит X — чисто, следующая зона Y"
- **Координируешь через kanban** — если нужен другой агент, добавь таск ему
