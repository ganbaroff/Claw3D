# Agile Rules — ZEUS Team
*Эти правила читают все агенты. CEO не объясняет дважды.*

---

## Как работает команда

**CEO (Юсиф)** — только стратегия. Добавляет таски в канбан, принимает деплои.
**Coordinator** — swarm.auto берёт таски из канбана и раздаёт агентам.
**Агенты** — работают по тасам, пишут выводы в memory/agent-findings/.

CEO не пишет код. CEO не объясняет как делать. CEO говорит ЧТО — команда решает КАК.

---

## Канбан правила (строго)

### Колонки
| Колонка | Смысл |
|---------|-------|
| 🔴 БЛОКЕРЫ | Стоп всего. Фиксить первым. |
| 🟡 В РАБОТЕ | Агент назначен. Работа идёт. |
| 🟢 BACKLOG | Приоритизировано. Ждёт свободного агента. |
| ✅ СДЕЛАНО | Готово. Не трогать. |

### Правила движения тасков
1. Таск попадает в BACKLOG с приоритетом (P0/P1/P2/P3)
2. Coordinator (swarm.auto) берёт P0 → P1 → P2 → P3
3. Назначает агента по колонке "Агент"
4. Агент пишет вывод в `memory/agent-findings/{task-id}.md`
5. Coordinator проверяет → двигает в СДЕЛАНО

### Размер таска
- Один таск = одна задача = одна встреча (≤2 часа работы агента)
- Большой таск → разбить на подтаски

---

## Sprint правила

- **Sprint = 1 неделя**
- **Sprint Planning** = обновить BACKLOG с приоритетами (CEO делает в воскресенье)
- **Daily Standup** = swarm.auto запускается каждые 4 часа (когда gateway online)
- **Sprint Review** = swarm.run "что сделано за неделю" → синтез для CEO

---

## Правила для агентов

### Что можно делать автономно (без CEO)
- Анализировать код и архитектуру
- Писать предложения и планы
- Находить баги и описывать фиксы
- Обновлять документацию
- Тестировать через API/CLI

### Что НЕЛЬЗЯ без CEO
- Деплоить в продакшен
- Мёрджить в main
- Менять env vars в Railway
- Удалять данные
- Писать от имени CEO куда-либо

### Формат вывода
Каждый агент после работы пишет файл `memory/agent-findings/{task-id}.md`:
```
# {TASK-ID}: {название}
**Агент:** {имя}
**Дата:** {дата}
**Статус:** Исследовано / Готово к деплою / Заблокировано

## Что нашёл
...

## Предлагаемое решение
...

## Файлы для изменения
- path/to/file.js: строки X-Y
...

## Что нужно от CEO
...
```

## Приоритеты

| P0 | Блокирует продакшен или безопасность. Делать сейчас. |
| P1 | Важно для пользователей. Этот спринт. |
| P2 | Улучшение. Следующий спринт. |
| P3 | Идея. Когда дойдут руки. |

|-------|---------------------|
